The Problem

Valuable signals, trapped in silos

📁

Data in every format

  • CSVs, Excel, PDFs, JSON, images, plain text — all in different shapes
  • No consistent schema, no single tool handles all of them reliably
💸

BI is expensive & rigid

  • Enterprise platforms require long setup, dedicated analysts, big budgets
  • Startups and lean teams can't justify the cost or time
🤖

Single-LLM approaches break

  • Generic prompting misses domain-specific signals and contradictions
  • No specialization = brittle reasoning on messy, mixed-format data

Decisions wait for data

  • Analysts manually clean and normalize before any insight can be drawn

Technical Deep Dive

Orchestrator runtime — how agents stay coherent

Agent Registry

  • Each agent declared as an AgentSpec: id, model tier, dependencies, responsibilities
  • Registry resolves CrewAI Agent objects at startup; exposes /agents/boot-status

Orchestrator Brain

  • Runs once after track classification using a heavy model
  • Emits a shared orchestrator brief and refines skip_agents
  • Core path agents (cleaning, aggregator, insights, exec summary) can never be skipped

Policy Engine

  • Retries with extra tolerance for rate limits
  • Time budget enforcement per stage
  • Stage gates (classify → ingest → process → aggregate → analyze → synthesize)
  • Adaptive behavior flags: sequential vs parallel execution

Agent Envelope Adapter

  • Every agent output normalized to a shared AgentEnvelope
  • Downstream agents consume structured envelopes, never raw LLM text
  • Enables swap of any agent without breaking the pipeline

Persistence & Replay

  • memory.json — cross-agent shared state
  • decision_ledger.jsonl — audit trail of every orchestrator decision
  • run_manifest.json — full run metadata and stage outcomes
  • pipeline_run_artifacts in Postgres for DB-level replay

Track Profiles

  • Each track defines an ordered stage pipeline resolved at run start
  • Agents not relevant to the track are cleanly excluded by the brain

Engineering Challenges

What was hard — and how we solved it

🔀

Multi-agent coordination

  • Strict dependency ordering to prevent circular waits and stalls
  • Explicit handoff contracts via the AgentEnvelope adapter
  • Orchestrator detects and recovers from stalled agent states
💾

VRAM-aware model selection

  • 12 GB VRAM floor across both dev machines
  • Heavy-tier vs light-tier split with explicit model assignment per agent
  • Featherless API as zero-VRAM alternative for remote inference
📄

Multi-format reliability

  • Six file types must normalize to a shared envelope before synthesis
  • Specialized processors per type with edge-case extraction handling
  • Gemini vision fallback for chart/table extraction from PDFs and images
📡

Real-time activity streaming

  • Pipeline logs must feel live without blocking heavy compute steps
  • Decoupled stream updates from core processing via per-step DB writes
  • Frontend polls run status and log tail; SSE is a planned upgrade
💰

Zero-cost constraint

  • External APIs used only where they provide outsized value (Gemini for OCR, ElevenLabs for audio)
  • Featherless for primary LLM traffic; Ollama for full local fallback
⏰

Two people, compressed timeline

  • Ruthless MVP scoping: core pipeline, optional stretch, future roadmap kept separate
  • Cross-domain pairing to maintain frontend + backend + agent coherence
Live Demo

5-step judge walkthrough

1

Open the app

Navigate to http://localhost:5173. Log in with demo@datalyze.ai / Demo@123. API docs available at http://localhost:8000/docs.

2

Configure company & track

Settings → Company: set company name, enable or disable public data scraping, choose the analysis track (e.g. Predictive Forecasting).

3

Upload data files

Upload → drop in a CSV, Excel, or PDF. Multiple files accepted. The file manager shows upload status, type detection, and size. Files are stored under a company-scoped path.

4

Start analysis & watch the pipeline

Dashboard → Start Analysis. Watch the live pipeline log as agents execute: classify → ingest → process → aggregate → analyze → synthesize → executive summary.

5

Explore the analysis detail page

View insight cards, SWOT analysis, trend charts, knowledge graph, orchestration diagram, and confidence indicators. Export PDF / CSV, or play the narrated summary. Use the AI chat for follow-up questions.

Bonus for technical judges: Settings → Developer — view agent boot status, DB health, run orchestrator smoke tests, and clear all analyses for a fresh demo run.
