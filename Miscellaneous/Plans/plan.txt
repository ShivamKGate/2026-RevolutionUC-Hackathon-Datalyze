1. THE IDEA — WHAT WE ARE BUILDING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

We are building a multi-agent AI-powered business intelligence platform
that allows any company to import their raw, messy, unstructured data —
hundreds of files across any format — and receive structured, actionable
business insights, strategic recommendations, forecasts, automation
suggestions, and visual analytics through an intelligent orchestrated
pipeline of AI agents.

The platform operates in two modes:
  - PRIVATE MODE: The company uploads their own internal data (CSV, PDF,
    Excel, JSON, images, etc.) and the system processes it entirely locally.
  - PUBLIC MODE: An admin-controlled web scraping agent collects publicly
    available data relevant to the company's industry and use case, which
    supplements or stands alone as the analysis source.

The key differentiator is the multi-agent orchestration architecture.
Rather than a single LLM trying to do everything, each agent has a
specialized responsibility, and a powerful orchestrator (running a heavy
reasoning model locally via Ollama) directs the entire pipeline — deciding
what to do next, which agents to fire, and when to loop for more data.

The entire application is LOCAL, ZERO-COST, and OPEN SOURCE. Anyone can
clone the repository and run it. There is no subscription, no cloud
dependency (except optional Gemini API calls, covered by hackathon credits),
and no rate-limited third-party lock-in.

4. THE AGENT ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

There are two types of LLMs in use:
  - HEAVY MODEL (orchestrator): Reasoning/thinking Ollama model (<12GB VRAM)
  - LIGHT MODEL (sub-agents): Fast, lightweight Ollama model

The system dynamically swaps between these as each agent is invoked.

--- AGENT ROSTER ---

1. ORCHESTRATOR AGENT (Heavy Model)
   The central brain. All other agents report to it. It decides which
   agent fires next, what data is needed, when to loop back for more,
   and when the pipeline is complete. It is also the home of the
   Strategic Recommendation Engine — a sub-module within the orchestrator
   that continuously proposes next steps to keep the pipeline flowing.

2. PIPELINE SELECTOR / GEMINI CLASSIFIER AGENT (Gemini API)
   First agent that runs after the user declares their use case. Uses the
   Gemini API to classify the pipeline track and configure which agents
   will be active. This is the critical decision gate of the whole system.
   Four primary pipeline tracks:
     a) Predictive / Trend Forecasting
     b) Automation Strategy
     c) Business Optimization
     d) Supply Chain & Operations
   Agents are reused across tracks with different configurations and priorities.

3. PUBLIC DATA SCRAPER AGENT (Light Model + Firecrawl)
   Scrapes publicly available data relevant to the company and selected
   pipeline track. Admin-controlled — admin can play/pause scraping.
   Only one company can be scraped at a time. Data is credible, relevant,
   and targeted based on the pipeline type chosen.

4. FILE TYPE CLASSIFIER AGENT (Light Model)
   Receives all uploaded files and identifies their type (PDF, CSV, Excel,
   JSON, image, plain text, etc.). Routes each file to the appropriate
   specialized processor agent. Also generates metadata tags for each file
   to help the aggregator.

5. SPECIALIZED FILE PROCESSOR AGENTS (Light Model + RAG for text-based files)
   One per file type:
     - PDF Processor: Uses a RAG pipeline to extract and chunk text
     - CSV Processor: Parses structured tabular data
     - Excel Processor: Handles multi-sheet workbooks
     - JSON Processor: Parses nested/flat JSON structures
     - Image/Multimodal Processor: Gemini Vision to extract charts, diagrams,
       handwritten notes, whiteboard photos
     - Plain Text Processor: Direct extraction

6. DATA CLEANING AGENT (Light Model)
   Cleans raw extracted text and structured data. Removes noise, fixes
   encoding issues, standardizes formats, and flags duplicate files
   (same file imported twice should not be processed twice).

7. SMART CATEGORIZER / METADATA AGENT (Light Model)
   Tags each processed file as: financial, HR, marketing, operations,
   sales, customer, product, etc. Provides this metadata to the aggregator
   as pre-classified context so the aggregator can work smarter.

8. AGGREGATOR AGENT (Heavy Model)
   The most critical data-layer agent. Takes all cleaned, classified data
   and decides:
     - Which data is highly useful
     - Which data is low priority
     - How different data pieces connect to each other
     - What the overall data story is
   Outputs structured data artifacts (chunks, datasets) ready for insight
   generation. Also builds the foundation of the Knowledge Graph.

9. CONFLICT DETECTION AGENT (Light Model)
   Cross-checks data sources for contradictions. If two files report
   different numbers for the same metric, it flags this as a warning
   in the dashboard. Helps surface internal inconsistencies — e.g., if
   a department is reporting different figures than another.

10. KNOWLEDGE GRAPH BUILDER AGENT (Light / Heavy Model)
    Builds an interactive visual knowledge graph of relationships between
    entities, departments, metrics, and data points across the entire
    dataset. Users can click on nodes to drill down into specifics.
    This is a core visual feature of the dashboard.

11. TREND FORECASTING AGENT (Heavy Model — toggled on/off by user)
    Detects time-series data and generates predictive forecasts for KPIs
    (revenue, churn, growth, etc.). Toggle in the dashboard. Outputs
    forecasting charts powered by D3.js / Plotly.

12. SENTIMENT ANALYSIS AGENT (Light Model)
    Analyzes any customer feedback, reviews, survey responses, or social
    data present in the uploaded or scraped data. Surfaces positive/
    negative sentiment trends.

13. INSIGHT GENERATION AGENT (Heavy Model)
    Synthesizes everything from the aggregator into human-readable,
    structured business insights. Decides what type of visualization
    to create (chart, table, graph) based on data type. Outputs:
      - Key insight cards with confidence score
      - Data provenance tags (which files generated which insight)
      - Chart/visualization specifications
      - Strategic recommendations

14. SWOT ANALYSIS AGENT (Heavy Model)
    Automatically generates a Strengths, Weaknesses, Opportunities,
    and Threats analysis from the combined data. One of the default
    output sections.

15. EXECUTIVE SUMMARY AGENT (Heavy Model)
    One-click generation of a board-ready executive summary of all
    insights. Clean, professional, exportable.

16. AUTOMATION STRATEGY AGENT (Light/Heavy Model)
    Based on the data, identifies where the company could implement
    automation. Generates a sample mini-prototype workflow description
    (2-3 step demo of what automation would look like for them), not
    the full automation itself.

17. DATA PROVENANCE TRACKER (System Layer)
    Not a separate LLM agent — a system-level tracker that tags every
    insight, chart, and recommendation with the exact source file(s)
    that contributed to it. Always visible in the UI.

18. NATURAL LANGUAGE SEARCH AGENT (pgvector + Light Model)
    Allows users to type a natural language question about their data
    and receive an answer grounded in the stored metadata. Also capable
    of generating charts on-demand from natural language requests.
    ("Show me a bar chart of monthly revenue by region.")

19. ELEVENLABS NARRATION AGENT (ElevenLabs API)
    Takes the executive summary and key insights and generates a
    podcast-style audio narration. Professional AI voice. Downloadable.
    This is the ElevenLabs integration for the prize category.

20. PRESAGE FOCUS AGENT (Presage SDK — OPTIONAL, SKIPPABLE)
    Tracks user focus/stress levels during data review. Does not affect
    the pipeline if not implemented or if it fails. Low priority.

21. SOLANA DATA LEDGER (Solana — OPTIONAL, SKIPPABLE)
    Immutable on-chain record of when data was ingested and which
    insights were generated — a lightweight audit trail. Completely
    skippable. Does not affect core pipeline.

5. PIPELINE FLOW — HOW IT ALL WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: ONBOARDING
  User lands on the app. They are asked:
    - Company name / context
    - What type of insights do you want? (Select: Predictive Forecasting /
      Automation / Business Optimization / Supply Chain)
    - Do you want public data scraped as well? (Yes/No)

Step 2: PIPELINE CLASSIFICATION (Gemini)
  The Gemini classifier agent reads the user's choices and configures the
  pipeline — which agents to prioritize, what data to look for, what
  insights to focus on.

Step 3: DATA INGESTION
  - User uploads files (any format, as many as they want — "one single
    import, in and out, finished")
  - File Type Classifier categorizes each file
  - Specialized processors extract raw text/data from each
  - (Optional) Public Scraper begins scraping in the background

Step 4: DATA CLEANING & CATEGORIZATION
  - Cleaning agent normalizes and cleans all extracted data
  - Metadata agent tags each file by domain
  - Duplicate detector flags repeated files

Step 5: AGGREGATION
  - Aggregator combines everything, scores usefulness, creates data chunks
  - Knowledge Graph builder begins mapping relationships
  - Conflict Detection agent flags inconsistencies

Step 6: ANALYSIS & INSIGHT GENERATION
  - Trend Forecasting (if toggled on)
  - Sentiment Analysis
  - Insight Generation (charts, tables, cards)
  - SWOT Analysis
  - Automation Strategy suggestions

Step 7: SYNTHESIS
  - Executive Summary generated
  - ElevenLabs audio narration generated
  - All insights tagged with data provenance

Step 8: DASHBOARD DISPLAY
  - Full dashboard renders with:
      * Insight cards (with confidence scores and source tags)
      * Knowledge graph (interactive)
      * Forecasting charts (if enabled)
      * SWOT section
      * Automation strategy section
      * Executive summary
      * Agent activity live feed (collapsible console)
      * Animated agent orchestration diagram
  - User can now chat with their data (natural language search)
  - User can request custom charts via chat

Step 9: EXPORT
  - Export full report as PDF
  - Export raw processed data as CSV
  - Generate shareable link (local network)
  - Download ElevenLabs audio narration

CORE CONCEPT:
A multi-agent AI-powered business intelligence platform. Companies import
their raw, unstructured data (any file format, any quantity) and receive
structured, actionable business insights, strategic recommendations,
trend forecasts, automation strategy suggestions, sentiment analysis,
SWOT analysis, and visual analytics — all processed through an orchestrated
pipeline of specialized AI agents running entirely locally.

Two operating modes:
  - PRIVATE MODE: Company uploads internal files; processed locally.
  - PUBLIC MODE: Admin-controlled web scraper collects public data
    relevant to the company's industry and selected use case.

The application is LOCAL, ZERO-COST, OPEN SOURCE. Runs from two terminals:
  Terminal 1: `npm run dev` (Next.js frontend + API routes)
  Terminal 2: `ollama serve` (local LLM server)

THE AGENT ARCHITECTURE:
There are 21 agents (some optional). Two LLM types: Heavy (orchestrator,
complex reasoning) and Light (fast, repetitive tasks). The system dynamically
swaps between them per agent invocation.

AGENT LIST:

1. ORCHESTRATOR AGENT (Heavy Ollama model)
   - The central brain. All agents report to it.
   - Decides which agent fires next based on current pipeline state.
   - Contains the Strategic Recommendation Engine sub-module.
   - Loops back to scraper or processors if more data is needed.
   - Manages state across the entire pipeline run.

2. PIPELINE CLASSIFIER AGENT (Gemini API — CRITICAL ROLE)
   - First agent to run after user onboarding inputs.
   - Classifies which of the 4 pipeline tracks to activate:
       a) Predictive / Trend Forecasting
       b) Automation Strategy
       c) Business Optimization
       d) Supply Chain & Operations
   - Configures which agents are prioritized and what data to look for.
   - Agents are reused across tracks with different configurations.
   - Also serves as Gemini Vision handler for chart/image extraction from PDFs.

3. PUBLIC DATA SCRAPER AGENT (Light Model + Firecrawl/Playwright)
   - Scrapes publicly available, credible data relevant to the company
     and selected pipeline track.
   - Admin-controlled: admin panel can play/pause scraping.
   - Only one company scraping session at a time.
   - Targets data relevant to the chosen use case (e.g., if automation
     track chosen, scrapes automation case studies and benchmarks).

4. FILE TYPE CLASSIFIER AGENT (Light Model)
   - Identifies file type of every uploaded file.
   - Routes each file to the appropriate specialized processor.
   - Generates metadata tags per file (domain, format, estimated content type).

5a. PDF PROCESSOR AGENT (Light Model + RAG pipeline)
    - Extracts and chunks text from PDFs using a RAG-style approach.
    - Handles scanned PDFs via Gemini Vision OCR if needed.
5b. CSV PROCESSOR AGENT
    - Parses structured tabular data, identifies column meanings.
5c. EXCEL PROCESSOR AGENT
    - Handles multi-sheet workbooks, extracts all sheet data.
5d. JSON PROCESSOR AGENT
    - Parses nested and flat JSON structures.
5e. IMAGE / MULTIMODAL PROCESSOR AGENT (Gemini Vision)
    - Extracts insights from uploaded images: charts, diagrams,
      whiteboard photos, handwritten notes.
5f. PLAIN TEXT PROCESSOR AGENT
    - Direct text extraction and chunking.

6. DATA CLEANING AGENT (Light Model)
   - Normalizes and cleans all extracted text/data.
   - Removes noise, fixes encoding issues, standardizes date/number formats.
   - Duplicate file detection: flags files that are the same to avoid
     repeat processing (same file imported twice → only process once).

7. SMART CATEGORIZER / METADATA AGENT (Light Model)
   - Tags each cleaned file as: financial, HR, marketing, operations,
     sales, customer, product, legal, etc.
   - Provides pre-classified metadata to the aggregator for smarter processing.

8. AGGREGATOR AGENT (Heavy Model — MOST CRITICAL DATA AGENT)
   - Combines all cleaned, categorized data.
   - Scores each data chunk for usefulness given the chosen pipeline track.
   - Prioritizes high-value data; deprioritizes but retains low-value data.
   - Creates structured data artifacts and chunks ready for insight generation.
   - Lays the foundation for the Knowledge Graph.
   - Makes final decisions about what the overall "data story" is.

9. CONFLICT DETECTION AGENT (Light Model)
   - Cross-references data sources for contradictions or inconsistencies.
   - Example: two departments reporting different revenue figures for same period.
   - Surfaces warnings in the dashboard as flagged insight cards.
   - Helps companies identify internal data quality issues.

10. KNOWLEDGE GRAPH BUILDER AGENT (Heavy Model)
    - Constructs an interactive knowledge graph of relationships between:
      entities, departments, metrics, time periods, and data points.
    - Output is a D3.js force-directed graph rendered on the dashboard.
    - Users can click on nodes to drill down into specifics.
    - One of the most visually impactful features.

11. TREND FORECASTING AGENT (Heavy Model — user toggleable on/off)
    - Detects time-series data patterns across uploaded files.
    - Generates predictive forecasts for key business KPIs.
    - Outputs forecasting charts (line charts, confidence intervals).
    - Rendered with D3.js or Plotly.
    - Toggle in dashboard settings panel.

12. SENTIMENT ANALYSIS AGENT (Light Model)
    - Analyzes customer feedback, reviews, survey results, social mentions
      present in uploaded or scraped data.
    - Outputs sentiment trend charts and summary cards.

13. INSIGHT GENERATION AGENT (Heavy Model)
    - Synthesizes aggregated data into human-readable business insights.
    - Determines appropriate visualization type per insight
      (bar chart, pie chart, line chart, table, heatmap).
    - Each insight includes:
        * Confidence score (% certainty based on data quality)
        * Data provenance tag (which source files generated it)
        * Plain-English explanation (click-to-reveal on dashboard)
    - Feeds insights into Executive Summary and SWOT agents.

14. SWOT ANALYSIS AGENT (Heavy Model)
    - Auto-generates Strengths, Weaknesses, Opportunities, Threats.
    - Rendered as a structured 2x2 grid on the dashboard.
    - Grounded in the actual uploaded data, not generic output.

15. EXECUTIVE SUMMARY AGENT (Heavy Model)
    - One-click generation of a board-ready executive summary.
    - Concise, professional, covers all key insights.
    - Exportable as part of the PDF report.

16. AUTOMATION STRATEGY AGENT (Light/Heavy Model)
    - Identifies areas where the company could implement automation.
    - Generates a 2-3 step sample prototype description of what
      automation would look like for their specific data context.
    - Does NOT build the full automation — demonstrates the concept.

17. DATA PROVENANCE TRACKER (System Layer — not an LLM agent)
    - System-level tracker embedded in the pipeline.
    - Every insight, chart, and recommendation is tagged with the
      exact source file(s) that contributed to generating it.
    - Visible in the dashboard as a clickable source tag on each insight.

18. NATURAL LANGUAGE SEARCH AGENT (pgvector + Light Model)
    - Enables users to query their entire dataset in plain English.
    - Uses pgvector for semantic similarity search over file metadata
      and processed data chunks.
    - Can generate charts on demand from natural language requests.
      Example: "Show me a bar chart of monthly revenue by region."
    - Powers the conversational data chatbot on the dashboard.

19. ELEVENLABS NARRATION AGENT (ElevenLabs API)
    - Takes the Executive Summary and key insight highlights.
    - Generates a podcast-style audio narration using ElevenLabs.
    - Professional AI voice, downloadable audio file.
    - Audio player embedded in the dashboard.
    - This is the primary ElevenLabs prize integration.

20. PRESAGE FOCUS AGENT (Presage SDK — OPTIONAL, SKIPPABLE)
    - Monitors user focus/stress levels during data review sessions.
    - Suggests breaks or highlights if user appears fatigued.
    - Does NOT affect the core pipeline. Can be disabled or omitted.

21. SOLANA AUDIT LEDGER (Solana — OPTIONAL, SKIPPABLE)
    - Creates an immutable on-chain record of when data was ingested
      and what insights were generated (lightweight audit trail).
    - Completely independent of the core pipeline. Skippable.

PIPELINE FLOW (step-by-step):

Step 1 — ONBOARDING:
  User is asked:
    - Company name and brief context
    - Use case / pipeline track selection (Predictive / Automation /
      Optimization / Supply Chain)
    - Whether to enable public data scraping (Yes/No)
    - Whether to enable trend forecasting (Yes/No toggle)

Step 2 — PIPELINE CLASSIFICATION (Gemini):
  Gemini classifier reads user inputs and configures the pipeline.
  Sets agent priorities and data targeting for the session.

Step 3 — DATA INGESTION:
  - User uploads all files via the upload interface (any format, any quantity).
  - Single import session — "import in, finished." One-time ingestion.
  - File Type Classifier categorizes each file.
  - Specialized processors extract raw content from each file type.
  - (If enabled) Public Scraper begins asynchronously in background.

Step 4 — CLEANING & CATEGORIZATION:
  - Cleaning agent normalizes and deduplicates.
  - Metadata agent tags all files by domain.

Step 5 — AGGREGATION:
  - Aggregator combines everything, scores usefulness, creates data artifacts.
  - Knowledge Graph builder begins mapping entity relationships.
  - Conflict Detection agent identifies contradictions.

Step 6 — ANALYSIS:
  - Trend Forecasting (if toggled on)
  - Sentiment Analysis
  - Insight Generation
  - SWOT Analysis
  - Automation Strategy suggestions

Step 7 — SYNTHESIS:
  - Executive Summary generated.
  - ElevenLabs narration generated from summary.
  - All insights tagged with data provenance.

Step 8 — DASHBOARD:
  Dashboard renders with all outputs. User can:
    - Browse insight cards, charts, SWOT, knowledge graph
    - Open agent activity live feed (collapsible)
    - View animated agent orchestration diagram
    - Chat with their data (natural language)
    - Request custom charts via chat
    - Toggle trend forecasting
    - Vote on insights (upvote/downvote)
    - Export report (PDF / CSV / shareable link)
    - Play ElevenLabs audio narration

Step 9 — EXPORT:
  - Full PDF report
  - CSV of processed data
  - Shareable local link
  - ElevenLabs audio download
